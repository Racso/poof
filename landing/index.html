// getting started

Up in minutes.

1
Install the binary
One curl. Works on any Linux server with Docker and Caddy already running.
2
Configure & start the server
Set your domain and auth token in /etc/poof/poof.toml. Then poof server.
optional: add a GitHub token to your config and Poof! will also automate GitHub Actions setup.
3
Register your project
poof add myapp registers the project, injects deploy secrets into your repo, and commits the CI workflow.
4
Push your code
Every push to main builds and deploys automatically. That's it — forever.
# 1. Install curl -fsSL https://raw.githubusercontent.com/racso/poof/main/install.sh | sh
# 2. Configure & start vim /etc/poof/poof.toml poof server &
# 3. Register a project poof add myapp ✓ Project registered · myapp.yourdomain.com ✓ Secret POOF_TOKEN set on github ✓ Workflow committed to .github/workflows/
# 4. Ship it git push ✦ https://myapp.yourdomain.com is live
No GitHub token? Not using GitHub at all?
  • Skip the GitHub token in your config — poof add still registers the project and configures routing.
  • Set up your CI pipeline manually in whatever tool you use.
  • Build your Docker image and push it to any registry the server can pull from.
  • Trigger deploys by calling POST /projects/myapp/deploy with the image tag.
# Build & push (any CI) docker build -t myimage:v1 . docker push myimage:v1
# Trigger deploy via API curl -X POST https://poof.yourdomain.com/projects/myapp/deploy \ -H "Authorization: Bearer $POOF_TOKEN" \ -d '{"image":"myimage:v1"}' ✦ https://myapp.yourdomain.com is live

Everything you need.

Nothing you don't.

◎
Zero DNS ops
One wildcard record covers every project, forever. Add a project, it's live at myapp.yourdomain.com instantly.
⚡
Automatic TLS
Caddy provisions ACME certificates automatically. Every subdomain gets HTTPS — no configuration required.
⬡
Per-project tokens
Each project gets an isolated secret. Compromise one, affect none. Generated and injected automatically via the GitHub API.
▤
Monorepo support
Multiple services in one repository with --folder. Workflows only trigger on changes to each subfolder.
⊙
One binary
Same Go binary runs as server daemon or CLI client. Zero runtime dependencies, SQLite state, single-file backup.
↩
Instant rollbacks
Something broke? poof rollback myapp redeploys the previous image in seconds from deployment history.
